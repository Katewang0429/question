個人簡介

我是Kate Wang, 目前在一間會計師事務所當助理, 主要負責中英文翻譯, 及官網內容更新,因此常常接觸Wordpress後台,
也有學過一些網頁基礎(ex. HTML, CSS)。平常會看Udemy上相關課程，並使用Visual Studio code 練習寫網頁。
有感於職涯和薪資發展考量及人生規劃,希望可以趁年輕一直學習新東西,不斷進步, 並且能藉由工作得到成就感。
所以思考後想花更多時間, 好好專注在一技之長, 未來能轉職成為前端工程師!
為什麼想成為工程師 ( 請以自我期許帶入：前端、後端、或全端工程師 ) 起因是在前一份工作,協助翻譯中英文的文件,但須更新至官網, 因時間急迫就上網查詢網頁相關資源, 最後將文件成功更新至官網, 也獲得小小成就感
(從發現問題並找到辦法解決的成就感)。而後開始找線上課程及相關資源,也覺得越學習越有興趣，期許自己成為前端 (同時也能了解一些後端)的工程師。
為了成為工程師，曾經做過什麼努力？ 初先在網路找資料自學(freecodecamp/ youtube),再來是五月參加了台大資工系統訓練的網頁設計基礎班, 有實際寫出一個簡單的Blog 及10月JavaScript網頁前端工程入門課程。
也已報名今年一月初, 有台灣人在澳洲的軟體工程師分享活動, 希望可以藉此更了解職場上的工作內容, 和實際會遇到的難題。
對於工程師的職業想像是什麼？ 需要長時間坐在電腦前debug, 一直解決各種不同的問題，也需要花時間和不同部門的人溝通協作
如果參與這個計畫，會怎麼安排學習時間？ 平日會利用下班時間2-3小時, 邊看線上課程(ex. Udemy, Coursera)邊練習寫code, 周末上午及下午各排出三小時空檔, 好好完成規畫的項目專案,
其他零碎的空檔, 我會看youtube上別人分享的website tutorial,學習他人遇到問題時不同的解決方式
如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？ 如果遇到寫程式卡關的部分,會先試著google 或在Stack overflow盡量找尋解答, 如果真的還不懂, 再找時間請教有經驗的工程師同事
想要對我們說的事情？ 感覺程式語言這領域變化很快，如何有效地持續精進專業能力, 符合現在職場需求?
